# Convergence Rules — 收敛判定与递归决策

> 知道"搜够了"和知道"搜什么"同样重要。
> 搜不够 → 答案片面。搜过度 → 浪费时间。

## 收敛判定框架

### 四个收敛条件（满足任一即停止）

```
Round N 完成 → 评估四个条件
    │
    ├── C1: 信息增益 < 阈值？        → YES → 收敛
    ├── C2: 所有子问题密度充分？      → YES → 收敛
    ├── C3: 最后一轮全是 LOW/UNVERIFIABLE？ → YES → 收敛
    └── C4: 时间/预算耗尽？          → YES → 收敛
    │
    └── 全部 NO → 未收敛 → 生成新查询 → 下一轮
```

---

## C1: 信息增益评估

**方法**：对比 Round N 和 Round N-1 的发现。

### 增益分级

| 增益级别 | 定义 | 判定 |
|---------|------|------|
| **MAJOR** 🟢 | 新发现显著改变了对问题的理解（推翻了之前的假设/发现了一个之前完全不知道的维度） | 必须继续 |
| **MODERATE** 🟡 | 新增了细节/数据/背景，但不改变整体结论 | 可继续（视 C2/C3 决定） |
| **MINOR** 🔴 | 只是新的引用/重复确认已知信息/略微不同的表述 | 接近收敛 |
| **ZERO** ⚫ | 连续两轮无任何新 HIGH 或 MEDIUM 发现 | 收敛 |

**操作规则：**
- MAJOR 增益 → 强制下一轮（进入新维度）
- MODERATE 增益 + C2 未满足 → 继续
- MINOR 增益 + C2 已满足 → 收敛
- ZERO 增益 → 立即收敛

---

## C2: 信息密度评估

对每个子问题，评估答案的"厚度"：

### 密度标准

| 子问题类型 | "充分"标准 | "不足"示例 |
|-----------|----------|----------|
| 事实类（"X 是多少"） | 有原始数据来源 + 至少 2 个独立验证 | 只有一篇转述文章的数字 |
| 机制类（"X 怎么工作"） | 有逐层解释 + 关键环节有引用 | 停留在"X 通过 Y 实现"而没有 Y 的原理 |
| 对比类（"X vs Y"） | 有并排对比 + 统一维度 + 各自独立数据 | 只有 X 的数据，Y 的数据是推测的 |
| 趋势类（"X 在变化吗"） | 有时间序列数据 + 方向一致 + 量级明确 | "X 在增长"但没有具体增速 |
| 操作类（"怎么做到 X"） | 有步骤 + 有前置条件 + 有常见坑 | 只有概括性建议 |

### 密度评分（每个子问题）

```
0-3 分（不足）：只有概括/少量信息，缺少细节
4-6 分（可接受）：有细节但某方面薄弱（如数据无原始出处）
7-10 分（充分）：关键细节 + 原始出处 + 交叉验证
```

**规则**：任一子问题 < 4 分 → 该子问题生成新查询。

---

## C3: 发现质量评估

**方法**：统计最后一轮新发现的置信度分布。

```
新发现 100% LOW/UNVERIFIABLE → 收敛（不是在找到答案，是在找到噪音）
新发现 > 50% MEDIUM → 检查是否解决了上一轮的空白
新发现任何 HIGH → 这轮有价值，但不是因为找不到更多——是因为这轮有新突破
```

**特殊情况**：
- "找不到"本身就是发现 → 标记为 `NEGATIVE_FINDING`，有信息价值
- 例："搜了所有主流学术数据库，没有找到支持 X 的 RCT"→ 这不是失败，这是发现

---

## C4: 时间/预算控制

| 模式 | 软上限 | 硬上限 | 超时后行为 |
|------|--------|--------|----------|
| Quick | 3 min | 5 min | 收敛，输出当前发现 + 标注"未穷尽，建议 Deep 模式" |
| Deep | 15 min | 25 min | 收敛，输出 + 标注未覆盖的维度 |
| Exhaustive | 40 min | 60 min | 如果主要维度已覆盖 → 收敛；否则标注暂停点 |

## 递归决策树（完整）

```
Round N 完成
    │
    ├── C1 = MAJOR → 【继续】生成新查询，关注新维度
    │
    ├── C1 = MODERATE →
    │   ├── C2 任何子问题 < 4 分 → 【继续】针对薄弱子问题
    │   └── C2 全部 ≥ 4 分 → 【收敛】
    │
    ├── C1 = MINOR →
    │   ├── C2 未读的源类型 > 2 种 → 【继续】补充缺失源类型
    │   └── C2 全部 ≥ 4 分 + 源覆盖 ≥ 4/5 → 【收敛】
    │
    ├── C1 = ZERO → 【收敛】
    │
    └── 已达 Round 5 → 【强制收敛】输出 + 标注未收敛原因
```

## 生成新查询的策略

当判定"继续"时，新查询不是重复上一轮的变体。需要更精准：

### 针对 C2 薄弱子问题

```
子问题 Q3 "DeepSeek 训练成本的具体构成" 密度 = 2/10
→ 已知：总成本 ~$5.6M
→ 未知：GPU 成本/数据成本/人力成本的拆解
→ 新查询：
  "DeepSeek V3 training cost breakdown GPU hours electricity"
  "DeepSeek 训练成本 构成 GPU 数量 训练时长"
```

### 针对缺失源类型

```
缺失：学术/论文源
→ 新查询（学术专用）：site:arxiv.org DeepSeek
→ 新查询（Google Scholar 风格）：DeepSeek-V3 technical report MLA attention

缺失：反对/质疑源
→ 新查询："DeepSeek cost criticism skepticism"
→ 新查询："DeepSeek 争议 质疑"
```

### 针对反向追问

```
已有发现："DeepSeek 使用 MoE 架构降低推理成本"
→ 反向追问："MoE 架构的成本优势在什么条件下成立？有没有不适用的情况？"
→ 新查询："MoE architecture limitations cost when NOT efficient"
```

## 收敛后标注

收敛后，在 Layer 6 报告中标注：

```markdown
## 收敛状态

- **收敛轮次**：3 轮
- **收敛条件**：C1=MINOR + C2 全部 ≥ 6 分 + 源覆盖 5/5
- **未覆盖维度**：无
- **如果有更多时间会搜什么**：[如果用户给更多时间，还有哪些方向值得深入]

## 或（未收敛时）

- **收敛轮次**：未收敛（Round 3，C4 时间上限触发）
- **收敛条件**：时间耗尽
- **未覆盖维度**：
  - [ ] 学术论文深度分析（只有摘要，未读全文）
  - [ ] 竞争对手的技术方案对比
- **如果继续搜**：[下一轮的查询建议]
```

## 特殊情况

### 信息真空

搜了所有能搜的源，仍然信息不足 → 可能不存在公开信息。

- 标记：`INFO_VACUUM`
- 报告中的处理：明确说"公开信息不足以回答这个问题的这个方面"
- **不是失败**——诚实标注信息边界也是答案

### 信息污染

搜索结果被 SEO 内容农场/AI 生成内容污染。

- 策略：缩小源范围（只搜权威域名）
- 必要时代价更高的获取方式（如直接访问特定网站而非搜索引擎）

### 信息矛盾

不同源给出相反的结论，且无法通过查找一手数据解决。

- 标记：`IRRECONCILABLE` 
- 报告中的处理：呈现双方论据 + 各自的证据强度 + 标注"当前无法判定"
- 矛盾本身是高价值信息——用户需要知道"专家们也在争论这个"
