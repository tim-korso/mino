# Cognitive Gap Analysis — 三层认知空白分析框架

> 从智谱调研中长出来的框架。解决一个核心问题：**为什么C端产品体验差的AI公司，可能在B端/技术端极其强大？我错过了哪些信号？**

## 核心洞见

**C端App体验是最容易获取的信息 → 也是最容易被过度加权的信息。**

当一个科技公司的核心产品不是C端App时，用App体验去判断公司价值 = 用餐厅甜点判断主厨水平。

## 三层分析框架

```
第一层：技术根基（权重最高，40%）
    ↓
第二层：商业验证（权重中等，35%）
    ↓
第三层：C端产品体验（权重最低，25%）
    ↓
认知空白 = 第三层信号 ↔ 第一+二层信号的落差
```

### 第一层：技术根基（权重40%）

核心问题：这家公司的技术是真实力还是PPT？

| 信号 | 为什么重要 | 去哪找 |
|------|-----------|--------|
| **自研架构/路线** | 不follow主流意味着能做差异化优化，长期不会被卡脖子 | 论文、技术博客、官方文档 |
| **开源策略与生态** | 开源 = 最强的开发者获客 + 技术自信的表现 | GitHub stars/downloads/forks |
| **论文产出与质量** | 有学术论文意味着技术不是黑箱调参 | arXiv、顶会、Google Scholar |
| **迭代速度** | 基座模型每2-3个月迭代 → 工程能力强，技术债不重 | 发布会记录、版本日志 |
| **全模态/全栈覆盖** | 完整性说明架构设计有前瞻性，不是单点突破 | 产品矩阵、官方文档 |
| **人才密度与来源** | 团队背景决定技术天花板 | LinkedIn、官网团队页、论文作者 |
| **被制裁/被管制** | 🚩 最强第三方背书：敌人帮你做了尽调 | BIS实体清单、出口管制新闻 |

### 第二层：商业验证（权重35%）

核心问题：技术能不能变成钱？谁在用脚投票？

| 信号 | 为什么重要 | 去哪找 |
|------|-----------|--------|
| **融资节奏与估值增长** | 资本是信息优势方的投票 | 招股书、天眼查、财经报道 |
| **投资方质量** | 一线VC/大厂/国资 → 专业尽调通过了 | 股东名单、融资公告 |
| **API/平台收入增速** | 最诚实的指标——开发者用脚投票 | 招股书、官方披露 |
| **客户结构与行业覆盖** | 高壁垒行业客户 = 技术被真正需要 | 案例研究、招股书 |
| **收入结构** | B端私有化 vs C端订阅 → 判断真正的商业模式 | 招股书（财报） |
| **研发投入强度** | 研发费用率 > 100% → 在烧钱做技术，不是烧钱做营销 | 招股书 |
| **海外收入/调用量** | 能在海外收钱 = 技术有全球竞争力 | OpenRouter、招股书 |
| **IPO进度/上市表现** | 公开市场的定价 | 港交所/科创板公告 |

### 第三层：C端产品体验（权重25%）

核心问题：普通人打开App看到的是什么？

| 信号 | 为什么容易误判 | 正确使用方式 |
|------|--------------|-------------|
| **App Store评分** | 大厂有专门的C端产品团队，创业公司没有 | 区分"产品体验差"vs"技术能力差" |
| **语音/对话体感** | 端到端语音所有公司都在早期 | 理解这个模态的技术成熟度曲线 |
| **功能完整度** | B端公司App是展示窗口不是核心产品 | 确认这个App在公司战略中的位置 |
| **UI/UX质量** | 设计资源投入 ≠ 技术能力 | 分开评估 |
| **用户口碑/社交媒体** | 容易被营销和情绪驱动 | 交叉验证 |

**C端降权规则：**
- 如果公司收入 < 20% 来自C端 → C端体验权重降至15%
- 如果公司核心产品是API/私有化部署 → C端App只是demo
- 如果公司被列入"AI六小虎"/创业公司 → C端打磨通常不如大厂

## 研究流程

### Phase 1: Quick Scout（快速侦察，5分钟）

```
1. 搜索"公司名 + 融资 + 估值" → 建立资本时间线
2. 搜索"公司名 + 模型发布 + 时间" → 建立技术迭代时间线
3. 搜索"公司名 + IPO + 上市" → 确认是否/何时进入公开市场
4. 搜索"公司名 + 制裁 + 实体清单" → 检查外部认可信号
5. 查看GitHub/OpenRouter/HuggingFace → 检查开发者生态
```

### Phase 2: Deep Dive（深度调研，15分钟）

对三层框架的每个信号做交叉验证：

1. **招股书分析**（如已上市）：收入结构、研发费用率、毛利率、客户集中度
2. **竞品对比**：在关键维度上与同行对比（不只是C端体验）
3. **时间线重建**：从成立到现在的关键节点，识别"信号何时出现但被忽视"
4. **认知空白计算**：对比C端叙事 vs B端/技术端现实

### Phase 3: Gap Report（输出认知空白报告）

输出结构化报告（见下方模板）。

## 批量扫描模式

当用户要求扫描"市场上未被C端正确认知的公司"时：

### 扫描范围
- 已上市科技公司（港股/科创板/A股）
- 即将上市的科技独角兽
- 重点关注：AI/半导体/机器人/生物科技/新材料

### 筛选条件
1. C端知名度低 或 C端口碑与其实际技术/商业能力不匹配
2. B端/技术端有显著壁垒
3. 有可验证的第三方背书（制裁/大客户/专利/论文）
4. 认知空白 > 3分（C端评分与技商平均差距）

### 输出
每家公司一个浓缩版分析（1-2段），包含：
- 公司名 + 一句话定性
- 认知空白是什么
- 关键信号
- 当前估值/市值（如上市）

## 组合跟踪模式（Gap Repair Portfolio）

> 核心逻辑：**认知空白本身不创造收益。空白被填补的过程才创造收益。**
> 认知效率低下的市场最终会修复错误定价——跟踪"修复进度"就是跟踪Alpha的释放节奏。

### 原理

```
认知空白 → 市场错误定价
    ↓
空白修复信号出现 → 定价效率提升
    ↓
空白缩小 → 超额收益释放
```

三层修复机制：

| 修复路径 | 触发条件 | 典型时间 | 案例 |
|---------|---------|---------|------|
| **财报修复** | 年报/半年报披露，财务数据被分析师覆盖 | 3-12个月 | 智谱上市后财报透明化，股价涨700% |
| **事件修复** | 制裁/大客户签约/技术突破被主流媒体报道 | 即时 | 寒武纪Q1扭亏为盈，市场重新定价 |
| **叙事修复** | C端产品体验改善 / 用户口碑逆转 / 社交媒体传播 | 6-24个月 | DeepSeek R1登顶App Store，一夜之间C端认知重置 |

- **40-60%**：空白正在闭合。财报连续验证，估值开始修复。
- **60-80%**：大部分已被定价。C端认知接近现实，超额收益窗口收窄。

### 组合管理规则

1. **加入条件**：认知空白 > 2.5分 AND 有可验证的修复催化剂在6-12个月内
2. **加注条件**：修复进度到达20-40%，且股价尚未充分反应
3. **减仓条件**：修复进度 > 70%，超额收益大部分已释放
4. **退出条件**：修复进度 > 85% OR 空白反向扩大（基本面恶化）
5. **审查频率**：每月一次完整重审，遇财报/重大事件即时更新

## 注意事项

- **这不是投资建议。** 框架识别的是"认知空白"——C端感知与B端/技术现实的差距。认知空白不代表股价会涨。
- **时效性敏感。** 框架分析基于当前时间点的公开信息。公司状态变化快，分析保质期约3个月。
- **区分"被低估"和"就是不行"。** 如果三层评分都低，那不是认知空白——那是真不行。
- **C端差 ≠ 技术好。** 反过来也成立：C端体验好不等于技术强（大厂可以用产品能力弥补技术差距）。
- **制裁信号是双刃剑。** 被制裁 = 技术被认可，但也 = 业务受限。要区分"技术认可型制裁"和"违规型制裁"。
- **修复不等于涨。** 认知空白修复(市场正确认知公司)和股价上涨之间有时滞，也可能已被提前定价。
