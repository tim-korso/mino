## 信号评分细则

### 第一层：技术根基（满分10分）

| 信号 | 满分 | 评分标准 |
|------|------|---------|
| 自研架构 | 2 | 2=全球独有架构；1=有差异化但在主流框架内；0=完全跟随 |
| 开源生态 | 2 | 2=全球下载>1000万+主流平台前10；1=有开源但影响有限；0=闭源 |
| 论文质量 | 1.5 | 1.5=顶会/顶刊多篇+被引高；1=有论文但影响力一般；0=无公开论文 |
| 迭代速度 | 1.5 | 1.5=<3月/版本; 1=3-6月; 0=>6月 |
| 全栈覆盖 | 1 | 1=覆盖4+模态；0.5=2-3模态；0=单一模态 |
| 人才密度 | 1 | 1=顶校/顶会作者密度高；0.5=一般；0=薄弱 |
| 制裁/管制 | 1 | 1=被列入实体清单等；0=无 |

### 第二层：商业验证（满分10分）

| 信号 | 满分 | 评分标准 |
|------|------|---------|
| 融资/估值 | 2 | 2=一线资本多轮+估值>200亿；1=有融资但规模小；0=未融资 |
| 投资方质量 | 1.5 | 1.5=大厂+国资+顶级VC；1=有其中1-2类；0=无知名投资方 |
| API/平台增速 | 1.5 | 1.5=年增>10倍；1=快速增长；0=无数据或低增速 |
| 客户结构 | 1.5 | 1.5=高壁垒行业+大客户；1=一般；0=C端为主 |
| 收入增速 | 1.5 | 1.5=>100%年增；1=50-100%；0=<50% |
| 研发投入 | 1 | 1=费用率>200%；0.5=100-200%；0=<100% |
| 海外验证 | 1 | 1=海外收入>10%或全球榜单前10；0.5=有海外存在；0=纯国内 |

### 第三层：C端产品体验（满分10分）

| 信号 | 满分 | 评分标准 |
|------|------|---------|
| App体验 | 4 | 4=行业顶尖；2=可用；1=有明显短板；0=很差或无App |
| 用户规模 | 3 | 3=>5000万MAU；2=1000-5000万；1=<1000万；0=无C端产品 |
| 口碑/评价 | 3 | 3=压倒性好评；2=一般；1=有明显负面；0=无C端存在感 |

### 认知空白计算

```
技术根基分 = sum(第一层各项)
商业验证分 = sum(第二层各项)
C端体验分 = sum(第三层各项)

技商平均 = (技术根基分 + 商业验证分) / 2
认知空白 = |C端体验分 - 技商平均|

解释：
- 认知空白 > 4: 严重错位（C端认知完全失准）
- 2-4: 显著错位
- 1-2: 轻微错位
- <1: C端认知基本准确
```

## 时间窗口识别方法论

"如果回到过去"分析的核心逻辑：

1. **列出C端用户在当时能接触到的信息**（App体验、媒体报道、社交讨论）
2. **列出专业投资者/开发者在当时能接触到的信息**（API文档、融资公告、GitHub、论文）
3. **找出二者之间的信息不对称**

关键问题：**当时有没有足够的信息让一个勤奋的研究者做出正确判断？**

如果答案是有 → 这是"认知懒惰"，不是"信息不对称"
如果答案是没有 → 这是真正的"信息不对称"，当时确实无法判断

## 常见误判模式

| 模式 | 描述 | 案例 |
|------|------|------|
| 甜点判主厨 | 用非核心产品判断公司 | 用GLM-4-Voice语音体验判断智谱的模型能力 |
| 时间错位 | 用一个时间点的体验判断快速迭代中的公司 | 2024年10月的体验判断2025-2026年的公司 |
| 大厂对比偏见 | 用字节/腾讯的C端产品标准判断创业公司 | 豆包vs智谱清言的体验对比 |
| 模态忽视 | 只看一个模态（语音）忽视其他模态（代码） | 不关心GLM在Code Arena的表现 |
| 制裁信号钝感 | 看不懂被制裁的正面含义 | 智谱被BIS列入清单 = 技术被美国政府认证 |
