# Source Hierarchy — 信息源五级金字塔

> 证据的置信度不仅取决于「什么证据」，还取决于「谁说的」。
> 同一条主张被 Consumer Reports 报告和被品牌官网声称，置信度天差地别。

## 五级金字塔

```
        ┌─────────────────┐
        │  Level A         │  独立系统测试机构
        │  → HIGH          │  方法论公开、可复现、非营利或订阅制
        ├─────────────────┤
        │  Level B         │  有方法论的职业测评
        │  → HIGH→MEDIUM   │  商业媒体但方法学透明
        ├─────────────────┤
        │  Level C         │  爱好者社区共识
        │  → MEDIUM        │  5+ 独立用户一致反馈
        ├─────────────────┤
        │  Level D         │  个人测评/体验
        │  → LOW           │  单一个人或小团体
        ├─────────────────┤
        │  Level E         │  品牌/渠道内容
        │  → LOW（自动降级）│  商业利益冲突
        └─────────────────┘
```

## 各级别详情

### Level A — 独立系统测试机构

**特征**：非营利或订阅制（不从厂商拿钱）、公开测试方法论、可复现、长期运营。

**代表机构**：

| 机构 | 覆盖品类 | 地区 |
|------|---------|------|
| Consumer Reports | 电子/家电/汽车/食品/… | 美国 |
| Stiftung Warentest | 电子/家电/食品/服务/… | 德国 |
| IIHS（公路安全保险协会） | 汽车碰撞安全 | 美国 |
| NCAP（各区域新车安全评鉴） | 汽车碰撞安全 | 欧洲/中国/日本/… |
| NSF International | 水处理/食品安全/… | 国际 |
| 中国消费者协会（比较试验） | 多品类 | 中国 |

**如何识别**：网站上有 "How We Test" 页面、不接受厂商广告、测试方法学可下载。

**置信度影响**：Level A 来源的证据 → 可锚定 HIGH（当证据类型本身足以支撑 HIGH 时）。

### Level B — 有方法论的职业测评

**特征**：商业媒体/公司，但测试方法学公开透明，有专业测试设备，长期专注品类。

**代表机构**：

| 机构 | 覆盖品类 | 地区 |
|------|---------|------|
| Wirecutter (NYT) | 电子/家电/家居/… | 美国 |
| rtings.com | 电视/显示器/耳机/… | 加拿大 |
| NotebookCheck | 笔记本/手机/平板 | 德国 |
| DXOMARK | 相机/手机摄像头/音频 | 法国 |
| DPReview | 相机/镜头 | 美国 |
| 老爸评测 | 母婴/日用品/食品 | 中国 |
| 消费者报道 | 多品类 | 中国 |
| 王自如/ZEALER | 电子产品 | 中国 |
| Car and Driver | 汽车 | 美国 |

**如何识别**：有测试设备照片、有量化数据、有评分体系、有批评内容（不全篇好评）。

**置信度影响**：Level B 来源 → 基础锚定 MEDIUM-HIGH。若与其他来源交叉验证一致，可升至 HIGH。

### Level C — 爱好者社区共识

**特征**：非专业人士但人数众多、无商业动机、集体经验形成共识。

**代表平台**：

| 平台 | 覆盖品类 | 特征 |
|------|---------|------|
| Reddit (r/BuyItForLife, r/malefashionadvice, r/buildapc, ...) | 全品类 | 按主题分区，高质量讨论 |
| 知乎 | 全品类（侧重中国语境） | 长文深度测评 |
| Bilibili | 电子/美妆/食品/… | 视频横评 |
| 什么值得买 (SMZDM) | 全品类 | 用户分享购买经验 |
| V2EX | 电子/数码/软件 | 程序员社区 |
| Chiphell | PC硬件/电子/汽车 | 高质量开箱和测评 |

**如何识别共识**：5+ 独立用户在不同时间/帖子中表达一致观点。单个帖子/单个用户的观点不是共识。

**置信度影响**：Level C 社区共识 → MEDIUM。人数越多、意见越一致、时间跨度越长 → 置信度越高，但上限 MEDIUM（无法排除群体偏见）。

### Level D — 个人测评/体验

**特征**：单一作者，可能有专业背景也可能没有，无系统测试方法论。

**代表**：单个 YouTuber、博客作者、淘宝/京东商品评价、小红书笔记、微博测评。

**置信度影响**：Level D → LOW。即使作者有专业背景，单人观察无独立验证。可作为发现产品选项的渠道，不做为验证主张的依据。

### Level E — 品牌/渠道内容

**特征**：有商业利益冲突——品牌官网、产品详情页、新闻稿、赞助内容、品牌自播间。

**置信度影响**：Level E → LOW（自动降一级）。品牌内容中即使引用了真实研究，也存在选择性引用风险——只说有利的不说无利的。

## 交叉验证规则

单一来源的权重受金字塔限制，但**多个独立来源交叉验证**可以突破：

```
单个 Level C 来源 → MEDIUM（上限）
3+ 独立 Level C 来源 + 观点一致 → 仍然 MEDIUM（无法突破）

Level C 共识 + Level B 测评 → 可升到 MEDIUM→HIGH
Level C 共识 + Level A 测试 → HIGH
Level D + Level D + Level D（多个独立博主） → 仍为 LOW（群体偏见风险）
```

**关键**：跨层级交叉验证比同层级重复更有效。一个 Consumer Reports 测试 + Reddit 共识 > 10 个 YouTuber 说同样的话。

## 信息源质量自检表

在 Phase 0.4 和 Phase 3 使用此表评估每个信息源：

```
□ 是否公开测试方法？        是 → +1 层级
□ 是否有商业利益冲突？      是 → −1 层级
□ 是否包含批评/负面评价？   是 → 信号增强（全好评 = 可疑）
□ 是否更新频率合理？        是 → 信号增强
□ 是否有专业资质/设备？    是 → +1 层级
□ 是否引用可验证的外部数据？ 是 → 信号增强
```

## 与证据类型的交互

信息源层级 × 证据功能类型 = 最终置信度：

```
Level A 来源 + safety_certification → HIGH（最高置信度组合）
Level A 来源 + brand_claim → LOW（来源可信但内容只是品牌自报，不能越级）
Level E 来源 + safety_certification → 按 safety_certification 本身的规则走
                                       （来源不可信但认证本身是独立第三方的）
```

**规则**：证据类型的权重由证据本身的「可验证性」决定，不由来源层级决定。来源层级用于降级——低层级来源的 `brand_claim` 比高层级来源的 `brand_claim` 更不可信。但 `safety_certification` 这种本质上来自第三方的证据，不会因为「从哪里听说这个认证」而降级。
