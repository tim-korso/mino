# Decision Matrix — 从置信度到购物决策

> 验证回答"真不真"，购物回答"该不该买"。这个映射不是自动的——需要引入偏好和场景。

## 四区决策

```
                可验证锚点多
                    │
      ┌─────────────┼─────────────┐
      │   🟢 放心区    │   🟡 减速区    │
      │              │              │
      │  "科学足够，   │  "靠谱但别急， │
      │   看价格就行"   │   试用装先试"   │
主观  ─────────────────┼──────────────────  主观
偏好低│                             │偏好高
      │   🟠 研究区    │   🔴 回避区    │
      │              │              │
      │  "信息不够，   │  "全是故事，   │
      │   自己去查"     │   别花钱"       │
      └─────────────┼─────────────┘
                    │
                可验证锚点少
```

### 各区的操作建议

| 区 | 条件 | 行动 |
|:--|------|------|
| 🟢 放心区 | HIGH≥3 + 无红灯 | 比较价格/渠道，下单。同一品类调成价格竞争 |
| 🟡 减速区 | MEDIUM 为主 + 无红灯 | 先拿试用装/小样。降低首次购买量。验证最关键的 1-2 个主张 |
| 🟠 研究区 | LOW 为主 + 有锚点但不够 | 自己去查缺失的关键信息。等更多独立测试结果。不急的品类 |
| 🔴 回避区 | FRAMEWORK 为主 + 有红灯 | 不买。信息基础不够做理性决策。情感驱动购买=自愿冒险 |

## 按产品类型的决策权重

不同类型，证据 vs 体验的权重不同。这个比例来自 Phase 0 的品质维度分析——当一个品类的品质维度主要客观可测（如补剂成分含量），证据权重就高；主要主观体验（如猫砂手感），体验权重就高。

| 产品类型 | 证据权重 | 体验权重 | 关键验证锚点 |
|---------|:--:|:--:|---------|
| 功效型护肤品 | 60% | 40% | 成分通用RCT + 该产品浓度检测 |
| 补剂/保健品 | 80% | 20% | GMP认证 + 第三方成分检测 + 重金属/微生物检测 |
| 食品 | 70% | 30% | SC编号 + 批次检测报告 + HACCP |
| 宠物食品 | 80% | 20% | 原料供应商 + AAFCO/FEDIAF 标准 + 批次检测 |
| 基因检测 | 90% | 10% | 测序技术参数 + 实验室认证(CLIA/CAP) |
| 防晒 | 70% | 30% | SPF/PA 检测 + 光稳定性测试 |
| 洁面/洗发水 | 40% | 60% | pH实测 + 表面活性剂类型 |
| 非功效日用品 | 30% | 70% | 第三方对比测试 + 售后政策 |
| **纺织品/服装** | 40% | 60% | 面料成分检测 + OEKO-TEX/GB 18401 + 水洗耐久性 |
| **电子产品** | 60% | 40% | FCC/CCC认证 + rtings/NotebookCheck 测试 + 电池安全 |
| **汽车** | 70% | 30% | NCAP/IIHS + Consumer Reports 可靠性 + 召回记录 |
| **房产** | 80% | 20% | 预售许可证 + 产权登记 + 开发商信用 |
| **家具** | 40% | 60% | GB 18580(甲醛) + 结构安全 + 实测舒适度 |
| 不粘锅 | 50% | 50% | 涂层类型 + 第三方耐刮测试 + PFAS检测 |
| 床垫/枕头 | 40% | 60% | 人体工学数据 + 试睡政策 |
| 猫砂 | 30% | 70% | 原料透明度 + 结团/除臭实测 |

> 新增品类（纺织品/电子/汽车/房产/家具）的权重由 Phase 0.2 的品质维度分析推导。当你面对不在表中的品类时，用同样的逻辑：品质维度有多少是客观可测的（成分/性能/安全）vs 主观体验的（手感/颜值/舒适度）？

## 决策前的"最后一次检查"

```
□ 锚点里有没有 HIGH 级别的主张？（至少 1 条）
□ 有没有红灯？（引用查不到 / 品类误识别 / 无产品测试只有成分研究）
□ 试用/退换政策是否降低了试错成本？
□ 现在缺的关键信息，对我个人是 dealbreaker 吗？
   ├ 是 → 不买，等更多信息
   └ 不是 → 可以买，但控制购买量
```

## 品类无关的决策逻辑

以上权重表覆盖了常见品类，但你总会遇到不在表中的品类。品类无关的决策规则：

```
1. 品质维度拆解 → 每个维度是「可客观测量」还是「靠主观体验」？
   ├ 客观多 → 证据权重高（≥60%）
   └ 主观多 → 体验权重高（≥60%）

2. 安全信号扫描 → 五维安全检查（化学/物理/生物/数据/财务）
   ├ 有 red 信号 → 直接 🔴，不继续
   └ 无 red 信号 → 继续评估

3. 证据层级映射 → 这个品类有什么认证/标准/测试机构？
   ├ 有 Level A/B 来源 → 可锚定 HIGH
   └ 只有 Level C/D/E → 上限 MEDIUM

4. 价格价值曲线 → floor / sweetSpot / diminishingReturns
   └ 推荐优先选 sweetSpot（每元买到最多品质）

5. 决策区判定
   ├ HIGH≥3 + 无红灯 → 🟢
   ├ MEDIUM为主 + 无红灯 → 🟡
   ├ LOW为主 + 有锚点但不够 → 🟠
   └ FRAMEWORK为主 + 有红灯 → 🔴
```

## 验证反馈整合

当 Phase 4 的验证反馈循环对关键主张做了外部验证后，决策需要相应调整：

| 验证结果 | 对决策的影响 |
|---------|------------|
| 外部源确认 HIGH 主张 | 维持 🟢 — 信心增强 |
| 外部源否定关键主张 | 该产品至少降一区（🟢→🟡→🟠→🔴） |
| 外部源无结果（不可验证） | 该主张降级，但不移动决策区（除非是安全相关） |
| 数字主张经 OCR 但未交叉验证 | 标注 `ocr_unverified`，影响该参数的权重 |
| 品牌自报数据被外部源否定 | red flag — 该品牌的其他主张也需怀疑 |

**核心规则**：验证反馈不是为了「证明我是对的」——是为了找出错误。如果验证后发现推荐需要修正，修正它。沉默的错误比承认的修正更糟。

- 这个框架解决的是"信息够不够做决策"，不是"这个产品好不好"
- 主观偏好（味道/肤感/颜值/品牌情感）不在框架覆盖范围内
- 紧急需求（"现在就需要一个枕头"）会自然降低证据要求——这是合理的

## 实战案例

### 案例：微基因 WGS（恋爱脑实验室 A10）

```
锚点：WGS技术描述（HIGH）+ 报告板块（LOW）+ 永久更新（LOW）
红灯：无独立第三方评估 + "永久免费更新"商业承诺不可靠
证据权重：90%（基因检测品类）
决策区：🟡 减速区

建议：技术描述准确（WGS vs 芯片），但下单前先查实验室认证 + 用户实际报告质量
```

### 案例：蓝盒子小蓝垫（AI Vision 版本）

```
锚点：0（AI Vision 编造的 WiFi/Zigbee 不是真实锚点）
红灯：品类误识别
决策区：🔴 回避区

需要 Tesseract 重新提取后再评估
```
