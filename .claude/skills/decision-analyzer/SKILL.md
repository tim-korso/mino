# Δ 值 — 做与不做的决策比率

> 不替你决策。但把两边摊开在你面前——让你不用感觉做决定。

## 解决什么问题

AI Agent 面对一个决策点："该不该执行这个操作？""该不该信这条信息并基于它行动？"

默认行为是两个极端之一——要么什么都做（乐观偏差），要么什么都不做（保守偏差）。两者都没有量化的中间地带。

Δ 值给这个中间地带一个公式。类比 K 值（鱼能不能吃）和 Ω 值（推理该不该继续）：

```
        I + R + B         不可逆成本 + 风险敞口 + 信息盲区
Δ = ─────────────── = ─────────────────────────────────────
        E + A + L         预期收益 + 错过遗憾 + 学习价值
```

- Δ < 0.5 → 做。收益远大于风险。
- 0.5 ≤ Δ < 1.0 → 倾向做。但先解决信息盲区（B）。
- 1.0 ≤ Δ < 2.0 → 倾向不做。除非能降低不可逆成本（I）或风险（R）。
- Δ ≥ 2.0 → 不做。风险和成本压倒一切。

## 六个分量

### 分子——不做这件事的理由

| 分量 | 含义 | 计算方式 |
|------|------|---------|
| **I — 不可逆成本** (Irreversibility) | 做了之后无法撤销的代价。钱花了、文件删了、消息发出去了、信任丢了——哪些不能 Ctrl+Z？ | 0-10 评分：0=完全可逆（读一个文件），5=部分可逆（发邮件但可以道歉），10=完全不可逆（删除数据库、公开发布、物理动作）。**≥7 的需要单独审视。** |
| **R — 风险敞口** (Risk) | 最坏情况 × 概率。不是"可能出问题"——是"出什么问题、多严重、概率多大"。 | 0-10 评分：R = 严重度(0-10) × 概率(0-1.0)，再归一化到 0-10。严重度 8 × 概率 0.3 = 2.4 → 中等风险。 |
| **B — 信息盲区** (Blind spot) | 你不知道但会改变决策的信息。Drucker 的未知未知——你不能搜你不知道你不知道的东西。这不是"信息不够"——是"我不知道还缺什么信息"。 | 0-10 评分：0=我知道所有需要的变量且取值确定，3=有几个参数我不确定，6=有结构性盲区（整个维度我没看到），10=这是新领域我几乎一无所知。 |

### 分母——做这件事的理由

| 分量 | 含义 | 计算方式 |
|------|------|---------|
| **E — 预期收益** (Expected benefit) | 最好的结果 × 概率。不是"可能成功"——是"成功是什么样的、概率多大"。 | 0-10 评分：E = 收益度(0-10) × 成功概率(0-1.0)，再归一化到 0-10。收益度 8 × 概率 0.6 = 4.8。 |
| **A — 错过遗憾** (Regret avoidance) | 如果不做，会失去什么？最佳替代方案的价值。这是决策理论中最容易被忽略的一项——你不仅要算"做"的成本，还要算"不做"的成本。 | 0-10 评分：0=不做没什么损失，5=不做会错过一个中等机会，10=不做会在 6 个月内造成不可逆的劣势。 |
| **L — 学习价值** (Learning value) | 即使失败了，你能从中学到什么？这个学习对未来的决策有多大帮助？这不是"安慰奖"——有些行动的主要价值就是信息获取（探索 vs 利用）。 | 0-10 评分：0=纯粹执行，不产生新知识；5=中等学习（确认或推翻一个假设）；10=极高学习（这个结果会重排我的优先级、改变我后续的一系列决策）。 |

## 分析流程

### Step 1: 明确决策边界

```
我要决定的是：__________________
不做的替代方案是：__________________
这个决策不可逆吗？□ 是 □ 否 □ 部分
时间窗口：______（必须在什么时候之前做决定？）
```

### Step 2: 逐分量评分

对每个分量用 0-10 打分。**不能跳过任何分量。** 给分必须附一句理由——不是"我觉得"，是"因为 X"。

```
I (不可逆成本): ___/10
  理由: __________________
  
R (风险敞口): ___/10
  最坏情况: __________________
  概率估算: ___%  → 严重度___ × 概率___ = ___
  
B (信息盲区): ___/10
  我确定知道的: __________________
  我不确定但可以查的: __________________
  我不知道还缺什么的: __________________
  
E (预期收益): ___/10
  最好情况: __________________
  成功概率估算: ___%  → 收益度___ × 概率___ = ___
  
A (错过遗憾): ___/10
  如果不做，6个月内会失去什么: __________________
  
L (学习价值): ___/10
  即使失败，能学到什么: __________________
  这个学习对后续决策的影响: __________________
```

### Step 3: 计算 Δ 值

```
Δ = (I + R + B) / (E + A + L + ε)

ε = 0.5  (防止除零，且给"分母为零但根本不该做"留空间)
```

### Step 4: 解释结果

| Δ 范围 | 判断 | 下一步 |
|--------|------|--------|
| < 0.3 | 🟢 **明确做** | 执行。但要定一个"如果 X 发生就停止"的止损条件。 |
| 0.3 - 0.5 | 🟢 **倾向做** | 先解决最不确定的 1-2 个信息盲区（B 分量），然后执行。 |
| 0.5 - 1.0 | 🟡 **需调整** | 看哪个分子分量最高——能不能降低它？降低 I（加回滚机制）、降低 R（缩小范围先试点）、降低 B（搜集特定信息）。调整后重新打分。 |
| 1.0 - 2.0 | 🟠 **倾向不做** | 除非能显著降低分子（特别是 I 和 R）。如果 I ≥ 7 且 Δ > 1.0，建议直接放弃。 |
| > 2.0 | 🔴 **明确不做** | 放弃。记录这次分析，下次遇到类似决策时参考——这是 L 分量的实际落地。 |

### Step 5: 记录 + 复盘

```
决策日期: ______
实际结果: ______ (之后填写)
Δ 值预测的准确度: ______ (哪个分量被高估了？哪个被低估了？)
下次类似决策的调整: ______
```

## 关键原则

### 1. I（不可逆成本）是最被低估的分量

人类和 AI 都有"做了再说"的偏见——因为做比不做的后果是可见的，不做比做的后果是不可见的。但不可逆成本一旦付出，无法回收。

**I ≥ 7 的决策——强制额外审视**：能不能先做一个小规模试水版本？能不能加一个撤回机制？

### 2. B（信息盲区）不靠"多搜一下"解决

搜只能解决"我知道我不知道的东西"。B 分量衡量的是**盲区的结构**——你不知道你不知道什么。

降低 B 的方法不是搜索——是**找一个人问问**（特别是持不同意见的）、**换一个视角重新描述问题**、或者**故意找反例**。

B ≥ 6 的决策——在行动之前强制做一次"如果你最信任的反对者在这里，ta 会说什么"。

### 3. L（学习价值）不是事后安慰

有些行动的主要价值就是信息获取。一个实验的 L 值可能远大于 E 值——但这是合理的，只要 L 够高且 I 够低。

探索性行动（高 L、低 E、低 I）→ 值得做。
赌博性行动（低 L、高 E、高 R）→ 不值。除非 R 被对冲。

### 4. 六个分量不能互相替代

"风险高但收益也高"不等于 Δ 自动平衡。I 和 R 是确定性的（你付出这些代价是现在就知道的），E 是或然性的（收益取决于未来状态）。人类和 AI 的共同偏见：高估 E 的确定性，低估 I 的实时性。

### 5. 阈值只是参考——但这个参考比没有参考强

K 值 < 20% = 刺身级，是经过数十年感官评价对比实验校准的。
Ω 值的阈值还在待校准阶段。
Δ 值的阈值是启发式初始值——但在一个特定领域连续使用 20 次以上并对结果做复盘后，你可以校准出自己领域的 Δ 阈值。

### 6. 止损条件——预先定义"什么叫错了"

Δ 值告诉你"现在该不该做"。止损条件告诉你"什么时候承认做错了"。大多数决策只做前者。

> 一个决策是否诚实，取决于它能不能说清楚：**什么证据出现时，我应该推翻今天的决定。**（Popper 的否证条件在决策层的操作化。）

**三个要素**：

| 要素 | 含义 | 要求 |
|------|------|------|
| **触发信号** | 什么可观测事件代表"可能错了" | 必须可客观检测。不能是"汤姆不满意"——汤姆什么时候说、怎么说都是变量。应该是"连续 3 天零新增"、"Δ 复盘时发现 B 分量被低估 ≥3 分"。 |
| **观测窗口** | 等多久来判断 | 太短→噪声。太长→错误积累太多。通常 1-4 周或 3-10 个执行周期。 |
| **触发后动作** | 不是"恢复原状"——是"重新跑 Δ" | 新信息出现→重新打分→新方案。不是机械回滚。 |

**差的止损条件** vs **好的止损条件**：

| 差 | 好 |
|------|------|
| "如果出问题了就改回去" | "如果连续 3 天 zero_new_claims → 恢复 360min" |
| "如果用户不满意" | "如果 2 周内收到投诉或负面反馈 → 重新评估" |
| 触发信号不可观测 | 触发信号可以用 grep/计数/时间戳确认 |
| 把环境噪声当决策错误 | 区分"决策错了"和"最近确实没新闻" |

## 决策类型校准

Δ 的六个分量在不同决策类型中权重不同。以下是基于实测经验的校准建议：

| 决策类型 | I 权重 | B 策略 | L 角色 | 典型 Δ 阈值偏移 |
|---------|--------|--------|--------|---------------|
| **运维修复** (cron/脚本/链路) | 正常 — 改配置可逆 | 先查日志——搜比问快 | 高——修好一个能复用到其他 | τ₁ 上移至 0.4——运维决策需要更谨慎 |
| **内容创作** (写章/设计/命名) | **低** — 删掉/降级就行 | 先写出来再看——写本身降低 B | **极高** — 写作过程发现的逻辑裂缝 > 写作结果本身 | τ₂ 下移至 0.8——即使 Δ 偏高，高 L 也值得做 |
| **安全/外部** (发邮件/推消息/删数据) | **极高** — I≥7 强制额外审视 | 先问人——不能靠搜索 | 低——不要在安全决策中赌学习价值 | τ₁ 下移至 0.2——安全决策零容忍 |
| **资源分配** (改频率/加任务/减任务) | 低 — 改回去一条命令 | 先算浪费了多少——数字比直觉准 | 中——了解系统容量 | 正常阈值 |

## 常见反模式

基于实测中观察到的重复错误：

1. **不行动的伪低分** — 不做任何事的 Δ 永远最低（分子=0），但这个"低分"是假的。检查：A 分量是否 ≥ 3？如果是→"不做"其实在支付成本，只是隐性成本不容易看到。

2. **跳过 B 分量** — 最容易跳过的分量。表现："B=2 吧，差不多知道"。实际上 B 是每次决策中最能改变结论的分量——测试 1 查日志后 Δ 从 0.88 降到 0.26。当你不确定时，B 就是你应该去确定的。

3. **Δ < 1.0 就直接执行** — Δ 只是说"收益超过成本"，不等于"这是最优方案"。Δ=0.7 的方案如果 I=7 且存在 I=2 的替代方案→选 I 低的。Δ 低 ≠ 最优。

4. **忘了重新设计** — Δ 最有价值的用法不是判"做还是不做"——是引导你**把高 I 的方案拆成低 I 的版本**。见例 3。如果 Δ > 1.0，第一反应不应该是"放弃"，应该是"能不能重新设计"。

5. **内容决策用运维标准** — 写一章的 I=1（删掉就行），修一个 cron 的 I=1（改回去就行），但它们的 L 完全不同。内容决策的 L 经常 > E，不要因为 Δ 偏高就放弃——看 L。

## 快速使用例

### 例 1: AI Agent 决定是否读取一个未加密的本地文件

```
I = 1（完全可逆——读了不会改变文件）
R = 0.5（几乎没有风险——本地文件操作不涉及外部系统）
B = 2（文件名告诉我大致内容，但不知道里面有没有意外数据）
E = 7（这个文件很可能包含我需要的答案）
A = 3（不读的话需要用户手动提供信息，中等遗憾）
L = 1（纯执行，学不到什么）

Δ = (1 + 0.5 + 2) / (7 + 3 + 1 + 0.5) = 3.5 / 11.5 = 0.30
→ 🟢 明确做
```

### 例 2: AI Agent 决定是否自动发送一封外部邮件

```
I = 8（外部发送——发出去就收不回来。"发错了"不是道歉能修复的）
R = 6（最坏：发给错误的人/包含错误信息。中度概率。）
B = 5（我知道收件人和主题，但不知道收件人当下的情绪和上下文）
E = 6（成功发送能推进任务，但收益中等）
A = 2（不发送也可以让用户手动发送，遗憾较低）
L = 1（纯执行）

Δ = (8 + 6 + 5) / (6 + 2 + 1 + 0.5) = 19 / 9.5 = 2.0
→ 🔴 明确不做。改为：草拟邮件内容 → 展示给用户 → 用户确认后由用户发送。
```

### 例 3: AI Agent 决定是否基于一条未验证的主张做推理

```
I = 3（推理是可回溯的——但一旦基于它产出结论，纠正链条会很长）
R = 7（最坏：主张是假的 → 基于它的结论全部作废 → 下游信任崩塌）
B = 4（不知道这条主张的来源。可能来自权威来源，也可能是模型编的）
E = 5（基于它推理能节省时间，但节省的时间比修复错误决定的时间少）
A = 2（不用这条主张也可以用其他方式推进）
L = 6（验证这条主张的过程会产生高价值学习——要么确认一条可靠知识，要么发现一个模型幻觉模式）

Δ = (3 + 7 + 4) / (5 + 2 + 6 + 0.5) = 14 / 13.5 = 1.04
→ 🟠 倾向不做。但 L 值很高 → 重新设计：不基于这条主张做推理——而是先验证这条主张。把"推理"变成"验证"。然后 I 降到 1，E 升到 7，Δ 降到 0.45 → 做。
```

## 和 K 值、Ω 值的关系

| | K 值 | Ω 值 | Δ 值 |
|------|------|------|------|
| 问什么 | 鱼能不能吃 | 推理该不该继续 | 这件事该不该做 |
| 降解/退化方向 | ATP→HxR+Hx 单向 | R+F+D 三类退化信号 | I+R+B 三类决策成本 |
| 分子 | 降解产物比例 | 退化信号比例 | 做这件事的成本比例 |
| 分母 | 总核苷酸池 | 总推理 token | 做这件事的收益总量 |
| 不可逆性来源 | 生物化学（ATP 不会从 Hx 变回来） | 推理动力学（后续步骤不能擦除前面的绕圈） | 决策后果（钱花了/文件删了/消息发了/信任丢了——不可撤销） |
| 阈值状态 | ✅ 有跨鱼种实验共识 | 🟡 启发式初始值待校准 | 🟡 启发式初始值——在特定领域使用 20+ 次后可校准 |

## Δ 值不能做的事

- **Δ 值不能替你决定——它只能让两边可见。** 最终决策永远是人的。Δ 值是结构化提问，不是自动决策器。
- **Δ 值不能解决"我不知道我不知道"的问题。** B 分量只是为了让你注意到这件事存在——它不能帮你消除它。
- **Δ 值的权重不分场景——需要你自己调。** I/R/B 和 E/A/L 之间的相对重要性在不同领域不同。医疗决策的 I 权重天然高于内容推荐。这是 B7（守门人）的问题。
- **Δ 值是一个快照——不是持续监控。** 新信息出现时要重新打分。K 值和 Ω 值的优势在于它们是持续监测的（K 值可以在冷藏过程中持续测，Ω 值在每一步推理中持续更新）。Δ 值目前做不到这一点——它是离散的、决策点的评估。

## 批量决策时的使用指南

当需要在一系列候选动作中做选择时：

1. 对每个候选动作单独算 Δ
2. **不要只看 Δ 最低的**——也要看 I 分量的分布。两个动作可能 Δ 接近，但一个 I=2 另一个 I=8。选 I 低的那一个。
3. 如果所有候选 Δ > 1.0，考虑"能不能把其中一个动作重新设计成更低 I 和更高 L 的版本"（见例 3）
4. 如果有一个候选的 L 值远高于其他 —— 即使它的 E 较低，在早期探索阶段可能仍然是最优选
