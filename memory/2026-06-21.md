## 智谱认知空白分析 + 技能封装 + 批量扫描

### 背景
汤姆发现自己对智谱存在"认知时间空白"——2024年10月下载智谱清言App，觉得语音通话不如豆包，判了死刑。2026年6月GLM-5.2发布时才发现这家公司已经非常强了。想知道：如果当时深入研究，能否提前发现智谱的价值？

### 验证结果
对上一轮分析的11个关键主张做了交叉验证，全部通过。关键数据：
- GLM-4-Voice：2024年10月发布（端到端语音，学术基准领先但体验不如豆包）
- 豆包实时语音：2025年1月20日发布（中文"断崖式领先"，满意度4.36/5 vs GPT-4o 3.18/5）
- 智谱被BIS列入实体清单：2025年1月15日（美国首次将中国大模型公司列入）
- GLM-4.6 Code Arena并列第一：2025年9-11月
- GLM-5.2：2026年6月13日发布，1M上下文，MIT开源，Design Arena第一
- 智谱港交所上市：2026年1月8日，"全球大模型第一股"

### 核心发现：五种认知空白模式

| 模式 | 描述 | 智谱案例 |
|------|------|---------|
| **甜点判主厨** | 用非核心产品判断公司 | 用GLM-4-Voice语音体验判断基座模型公司 |
| **时间错位** | 用一个时间点的体验判断快速迭代中的公司 | 2024年10月→2026年6月，跨越至少6代基座模型 |
| **大厂对比偏见** | 用大厂C端标准判断创业公司 | 豆包(字节) vs 智谱清言的体验对比 |
| **模态忽视** | 只看一个模态忽视其他 | 不关注GLM在Code Arena的表现 |
| **制裁信号钝感** | 看不懂被制裁的正面含义 | 智谱被BIS列入清单=技术被美国政府认证 |

### 产出
1. **技能**：`cognitive-gap-analysis` — 三层框架（技术根基40% + 商业验证35% + C端体验25%）+ 研究流程 + 输出模板
2. **DeepSeek分析**：认知空白3.75分（比智谱更大），核心错位——C端以为的"免费AI聊天App"实际是"全球顶级开源基础模型研发实验室"
3. **公司扫描**：8家认知空白公司，Top3 第四范式(完全隐形)、商汤(时间错位)、寒武纪(拐点已过但C端未更新)

### 关键教训
- C端App体验是最容易获取的信息→最容易被过度加权
- 真正有价值的信号（自研架构、API增速、被制裁、投资方质量）不在App里
- 被美国制裁是中国AI公司最硬的外部技术认证
- DeepSeek的"三不铁律"打破被C端误读为"向资本低头"——实际是用资本来抵御资本

### 🧭 探路：认知空白分析框架的设计过程

**出发点**：汤姆问"如果当时深入研究语音，会更好看智谱吗？"→ 答案是不会，因为语音维度的研究只会让他更坚定地判死刑。这暴露了问题不在于"研究不够深"，在于"研究维度不对"。

**过程**：
1. 先用claim-verification验证智谱分析的关键主张（全通过）
2. 从智谱案例中抽象出三层框架——第一层（技术根基）和第二层（商业验证）的信号完全在C端App之外
3. 发现"被制裁"是一个被严重低估的正面信号——美国政府帮你做尽调
4. 用DeepSeek做交叉验证——发现DeepSeek的认知空白比智谱更大（3.75 vs 2.8），但模式不同（主动沉默 vs App体验差）
5. 批量扫描确认框架通用性——第四范式/商汤/寒武纪/百济神州/优必选各有不同误判模式

**教训**：
- C端体验差≠技术不行——在AI基座模型领域尤其成立
- 框架的评分维度需要随公司类型调整权重（B端公司降C端权重）
- "被制裁"信号需要区分"技术认可型"和"违规型"
- 时间错位是最普遍的误判模式——科技公司的迭代速度远超C端认知更新速度
