### Workflow Watchdog + API 断连深入分析

针对 write 管线中反复出现的 Agent stall 和 Connection closed 问题，做了跨 602 个 Agent 的全量统计分析：

- Agent stall (180s 无输出): Claude Agent SDK Bun runtime 硬编码。长 Agent（写章/对抗充实的研究阶段）3-5 分钟沉默触发。Resume 可借已完成 Agent 的缓存绕过。
- API 断连 (Connection closed mid-response): DeepSeek v4-pro 流式连接中断。比 stall 更致命——首次无缓存、每次重试从头烧 token。602 个 Agent 中仅 3 个 journal 级错误 (0.5%)，但 Workflow 级别有 6 次中断。
- Pipeline 天然容错：断了 A 继续跑 B，resume 时 A 重试 B/C/D 秒出缓存。单 Agent 大任务最危险。
- 排查结论：MyAgents 无 watchdog 配置暴露；SDK 硬编码 180s/6 retries。规避方式 = 大 Workflow 拆小 + 长 Agent 改后台独立跑。
