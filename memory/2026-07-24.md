## PPT COM v4 — Cinematic Design System — 13:40 CST

交付 `watchlist-week6-v4.pptx`: 14页重构 (文字墙→卡片化) + 235个现代入场动画 (TimeLine.MainSequence) + 14页演讲者备注 + 图表直写修复。

**QA 抓出的 v3 三个隐藏 bug (全部修复)**:
1. **BGR 字节序** — COM `Color.RGB` 是 BGR packed int, v3 全甲板"品牌红"实际渲染成蓝紫。0xD33941 → 红需传 0x4139D3。视觉 QA 子代理一眼抓出, v3 从未视觉验证过
2. **PS 双引号 `$` 变量展开** — v3 的 `$31.5B`/`$1B`/`$100M` 全部被 PS 展开成空串, 金融数据金额丢失。v4 全部单引号字符串
3. **图表嵌入 Excel 失败** — v3 图表存的是默认示例数据 (系列1/2/3)。v4 改 `SeriesCollection.Values/XValues` 直写, 绕开 ChartData.Activate

**🧭 探索: COM 动画探针**
出发点: "令人惊叹的动画"需要 TimeLine 模型, 但 office.ps1 只有旧版 AnimationSettings, MsoAnimEffect 枚举文档 404。
过程: 写探针脚本对 96 个 effect ID × 4 个 trigger 值逐一 AddEffect → 存盘 → 解包 pptx XML 读 presetID/presetClass/nodeType。XML 是唯一事实来源: trigger 1=clickEffect/2=withPrevious/3=afterPrevious, 4=OnShapeClick (MainSequence 不支持直接报 Invalid request)。特效 ID 1-53=入场, 54-82=强调, 86-95=路径。
教训: **枚举值别猜, 探针+XML 取证一次到位**。DisplayName 返回形状名 (本地化) 无用; EffectType 原样返回 (晚绑定不校验枚举) 也无用——只有落盘 XML 说真话。

**🧭 探索: 级联节奏模型**
第一版: AfterPrevious + 累积 delay → 总时长 = 全部 duration + 全部 delay 线性叠加, S11 横幅会被自动换页切掉。
正解: 首效果 AfterPrevious, 后续全部 WithPrevious + 恒定 step → 相对前一效果 START 的均匀交叠级联。2秒完成入场, 剩余时间静态阅读。
教训: afterEffect 的 delay 是相对前一效果 END, withEffect 是相对前一效果 START。想"流动感"只能用 withEffect 链, afterEffect 链必然是串行的。

**其他坑**: Font.Size 只接受 [float], double 直接 InvalidCastException (int 可以); here-string 作实参时 `'@` 收尾行不能跟参数; 多行函数调用在 @(...) 闭合后换行即断。
