---

## Rules for Session Manifests

1. **Create one at the end of every session** where design exploration or significant decisions happened
2. **Don't create for purely conversational / Q&A sessions** — only when something was built or decided
3. **Link to topic files, not the reverse** — session manifests are the glue between temporal navigation and topic navigation
4. **Keep Pending items actionable** — they become the starting point for the next session
5. **Include Design Artifacts section** — image paths, sketch descriptions, anything visual that won't survive in plain text
