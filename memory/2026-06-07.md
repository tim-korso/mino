### pqa-app React 重构 + 部署 (20:00-20:25)

汤姆反馈 pqa 项目出错。诊断发现问题链：
1. 🔴 `index.html` L712 多余 `}` → `SyntaxError` → App 白屏（鲁蛇 AI 修复 Q-PQA bugs 时引入）
2. 🟡 `data.js` 766KB 同步加载 → 阻塞首屏
3. 🔴 重构后 `const` 声明不挂 `window` → React 组件取不到数据 → 内容加载空白

**修复 + 重构：**
- 删多余 `}`，加 `defer`，JS 语法验证通过 → 快速修复版部署 iPhone
- 汤姆要求「用 Capacitor 重构」→ 标准化工程迁移：
  - 51KB 单一 HTML → React + Vite + Capacitor（7 组件 + App.css + markdown.js）
  - `data.js` 保留为 public/ 静态资源，加桥接脚本解决 `const` → `window` 问题
  - `npm run build` → `npx cap sync ios` → `xcodebuild` → `xcrun devicectl` 全链路通过
- 部署时 iPhone 锁屏导致 launch 失败，安装成功——解锁后点图标即可

**关键教训：**
- 非模块脚本的 `const` 不创建 `window` 属性（只有 `var` 做这事）→ 迁移到 ES module 时需显式桥接
- Capacitor SPM 首次拉依赖可能 GitHub 超时，重试即可（非代码问题）
