## book-figure：VLM 定位管线——从踩坑到原理

### 🧭 探索：字母锚点 → 圆点锚点 → VLM 定位

**出发点**：book-figure 的自动标注一直不稳定。轮廓极值点、凸包缺陷、曲率分析都试过——各有各的失效模式。想用 AI 自己在图上放标记，我们检测标记位置然后替换成中文标签。

**过程**：

1. **字母锚点**：prompt 里加 "write letter A next to the ear" → 连通分量检测 → 16 个候选，大部分是竖长条轮廓碎片。AI 画的"A"和轮廓线上的折线没区别。看了原图确认：扩散模型画不出可识别的文字。

2. **圆点锚点**：以为问题是"字母太复杂"，简化成小黑圆点 → HoughCircles 抓了 62 个"圆"——全是牛身体曲线弧段。圆点在扩散模型里同样和轮廓线混在一起。

3. **VLM 定位**：回看原理。扩散模型从噪声中迭代去噪，整张图同时浮现——没有"在 X 位置放置 Y"这个操作。而 VLM 的训练目标就是看图→定位语义区域。两个模型的能力边界是互补的，不是竞争关系。

管线：通义万相生纯线稿（不要求任何标注）→ Qwen-VL 看图定位部位 → SVG 标注叠加。牛 4/4 部位一次成功。

**教训**：匹配模型类型到任务。不是"哪个模型厉害"，是"哪个模型的训练目标对上了这个任务"。扩散模型做生成、VLM 做定位——分而治之。

这条规则和已有的 "LLM=Scout≠Judge"、"构建者不能验证自己的输出"是同一条元规则的不同实例。

## see-clearly：从经验到六条经典原则

### 诞生

汤姆提了四步框架：生成原理→识图原理→问题分解→部件联系。娜娜把它操作化成 `/看清` skill。

### deep-research 完善

搜到六条经典原则作为思想渊源：Separation of Concerns (Dijkstra 1974)、Do One Thing Well (McIlroy 1978)、Rule of Composition (Unix)、Rule of Separation (Unix)、Gall's Law (1975)、Law of the Instrument (Kaplan 1964/Maslow 1966)。

加了一个完整 Workflow 分析示例——四问如何发现 pipeline()≠Unix pipe、schema≠内容验证、null 丢失失败语义。

### 实测

应用到三个非工程问题：牛奶有害论、炒股有害论、smmart 下载效率。框架在非技术领域同样有效——"把不同子问题压缩成一个词"是通用反模式。

## agent-orchestration：Workflow 机制边界修复

`/看清 Workflow` 发现四个边界越界：
1. pipeline() 名不副实（是 per-item stream 不是 Unix pipe）
2. parallel() 被名字吸引滥用
3. schema 只验结构不验内容（合并幻觉）
4. null 丢失失败语义

落地 6 处改动：
- pipeline() 加 ASCII 对比图
- 新增 `safeAgent()` error wrapper
- 新增 Pattern 7: Verify & Recover（语义验证+条件重试）
- 新增 Pattern 8: No-silent-caps
- 新增"何时不用 Workflow"反模式指南（5 场景+替代）
- 常见坑 8→10 条

## smmart：三层管线 + 全矩阵实测

### 看清分析

smmart 的瓶颈不是"源不够"也不是"工具不行"——是把三种不同机制的源（直接 URL、API 协议、云盘登录）用同一种方式（Agent 手动搜）处理。

### 优化落地

1. `smmart-search.py` — 多源并发搜索脚本，Agent 不参与搜索
2. `smmart-workflow.js` — Agent 工具自举 Workflow（检测→搜索→下载→验证）
3. SKILL.md 三层管线重构

### 全矩阵实测 (2026-07-15)

12 类资源 × 22 源可达性测试：
- ✅ GitHub raw / YouTube / haxmac.cc / 423down.com / ghxi.com / maoken.com / Wallhaven / Unsplash / libgen.li / Z-Lib / Sci-Hub .st
- ❌ LibGen.is / Anna's Archive / 鸠摩搜书 / xmac.app / LibriVox / alipansou.com / 1337x / KHInsider

工具全部就绪：aria2c 1.37 / yt-dlp 2026.07 / gallery-dl 1.32 / ffmpeg 8.1 / spotdl 2.2.2 / imagemagick 7.1.2

### 关键发现：libgen.li 换 Bootstrap 新版面

旧 HTML 解析器（valign=top 表格）对 libgen.li 失效。新格式用 Bootstrap卡片布局，index.php 而非 search.php，下载通过 IPFS 网关而非 library.lol 直链。smmart-search.py 需要更新解析器。
