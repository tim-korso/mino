# Cognitive Gap Analysis — 三层认知空白分析

> 2026-06-21 创建，从智谱调研中生长。

## 核心框架

识别被C端大众低估或错误认知的科技公司。三层分析：

```
第一层：技术根基（权重40%）
    ↓
第二层：商业验证（权重35%）
    ↓
第三层：C端产品体验（权重25%）
    ↓
认知空白 = |C端评分 - 技商平均|
```

## 五种误判模式

| 模式 | 描述 | 发现于 |
|------|------|--------|
| **甜点判主厨** | 用非核心产品判断公司 | 智谱（语音App vs 基座模型） |
| **时间错位** | 用旧体验判断快速迭代的公司 | 智谱（2024.10→2026.06，6代模型） |
| **大厂对比偏见** | 用大厂C端标准判断创业公司 | 智谱（豆包 vs 清言） |
| **模态忽视** | 只看一个模态忽视其他 | 待验证 |
| **制裁信号钝感** | 看不懂被制裁的正面含义 | 智谱（BIS清单 = 技术认证） |

## 关键信号清单

### 第一层（技术根基）：自研架构、开源生态、论文、迭代速度、全栈覆盖、人才密度、**被制裁**
### 第二层（商业验证）：融资节奏、投资方质量、API增速、客户结构、收入结构、研发投入强度、海外收入
### 第三层（C端体验）：App评分、语音/对话体感、功能完整度、UI/UX、社交口碑

## 已完成分析

| 寒武纪 | 严重 | "亏十年" vs 2026Q1已盈利+国产替代刚需 |
| 百济神州 | 显著 | "烧钱药企" vs 泽布替尼72国放量 |
| 优必选 | 显著 | "跳舞玩具" vs 14亿工业订单 |
