# 购物比价

> 消费者侧 AI 购物助手的可行性研究——技术方案、现有工具、平台限制

## 状态
**调研阶段** — 2026-06-03 完成初步调研，结论：不需要自建，现成 App 足够；自建的话官方联盟 API 是正确路径

## 核心结论

### 消费者比价：别造轮子
- **慢慢买**、**什么值得买**、**喵喵折**、**购物党** 四款 App 覆盖了 99% 的比价需求
- 慢慢买最强（180 天历史价曲线、30+ 平台、准确率 99%）
- 组合：慢慢买（大件查底价）+ 什么值得买（看评测攻略）+ 喵喵折（小程序随手查）

### 自建方案：官方联盟 API
如果真要做「AI Agent 帮你比价」：
- **京东联盟**：个人可申，商品搜索+实时价格+佣金
- **多多进宝**：个人可申，拼多多官方商品 API
- **淘宝联盟**：需企业资质，门槛最高
- 这些 API 免费、合法、稳定——远好于爬虫

### 爬虫方案：三重壁垒不可逾越
1. **技术**：淘宝/拼多多的 AI 行为识别（浏览器指纹、鼠标轨迹、滑块验证）
2. **法律**：2025 最高法判例，绕过反爬商用 = 不正当竞争，判赔 500 万
3. **成本**：住宅代理 IP 池 + 账号池 + 持续维护，TCO 远超买 SaaS

## Zyte 评估

| 维度 | 结论 |
|------|------|
| 技术能力 | 强——代理池+浏览器渲染+320K+ 反爬策略，欧美网站成功率 98% |
| 中国电商 | **弱**——无淘宝/京东/拼多多公开案例，代理池缺中国住宅 IP |
| 价格 | $5 试用 → $100/月起，中国电商大概率走 "hard" tier |
| MCP 集成 | 有（`apscrapes/zyte-fetch-page-content-mcp-server`） |
| Claude Code 插件 | 有（`claude plugin install zyte-web-data@zyte-ai`） |
| 建议 | 不适合中国电商比价；适合抓取欧美公开网页内容 |

## mcp-bijia 评估

- npm `mcp-bijia@0.1.0`, MIT, 作者 eesire
- 3 工具定义：search_products / get_lowest_price / get_affiliate_link
- **现状**：骨架代码，所有搜索函数返回 stub（`available: false`），需要接入真实联盟 API
- 潜力：如果填完，是 AI Agent 比价的最佳入口（直接走 MCP 协议）
- 需要：TAOBAO_APPKEY / JD_APPKEY / 对应的 SECRET

## 可用工具链

### AI Agent 可用的浏览器
- Playwright MCP + Chrome（`brew install --cask google-chrome`）
- 能打开小红书首页，能看推荐流
- 搜索受限需要登录态

### 第三方电商数据服务
- 鼎点数据：品牌控价 SaaS
- JustOneAPI / DataCart：淘宝/闲鱼/拼多多数据 API（几千到几百/月）
- RapidAPI Taobao DataHub：有免费额度
