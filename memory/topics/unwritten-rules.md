## 核心概念

**潜规则是从正式制度的裂缝里长出来的。** 不是文化，不是道德败坏——是给定制度结构下的理性均衡。

任何一个域（政治/商业/平台/饭局/职场/物业/购物/法律……）都有正式规则和实际运行规则之间的裂缝。引擎的任务不是"知道答案"——是"知道怎么结构性地找到答案"。

## 五步分析框架

1. **定位正式规则与裂缝** — 明文规定 + 自由裁量空间 + 信息不对称 + 监督成本
2. **识别潜规则形态** — 利害计算 + 合法伤害权来源 + 系统性/个例性
3. **判断规则切换信号** — 吴思四级模型 + 运动式执法 + 政治罪信号
4. **位置分析** — 黄仁吾的七面镜子——用户在制度结构中的位置和压力方向
5. **行动建议** — 戚继光困境 + 最低有效剂量 + 止损条件

## 竞争壁垒

1. **框架是原创IP** — 7章传导链来自用户自己的书，任何竞品没有这个理论深度
2. **跨域通用** — 政治/商业/平台/饭局/职场/物业/购物——框架放任何域都管用
3. **已验证质量差异** — 框架引导 vs 裸 LLM 对照测试（抖音案例），框架多了权力分析/合法伤害权/规则切换/位置分析
4. **CLI-native** — prompt 可管道到任何 LLM
5. **实时搜索** — 域知识动态获取，不依赖预先积累
