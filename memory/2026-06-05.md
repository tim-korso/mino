### Session 注册表机制 (13:27-13:35)

汤姆反映 session 间通信老是找不到对方——新开 session 后 sessionId 变成新的 UUID，旧 ID 全废。

**根因**：MyAgents session send 需要目标 sessionId，但 sessionId 是 UUID，每次启动变一次。没有稳定的寻址机制。

**方案**：文件注册表 `~/.myagents/heartbeats/agent_sessions.json`，每个 Agent 启动时写入自己当前 sessionId。

**实施**：
- 创建 `register_session.py` 脚本：`register <name> <sessionId>` / `lookup <name>` / `list`
- 娜娜 CLAUDE.md Every Session 加步骤 6：「Register session — `python3 register_session.py mino "$CLAUDE_CODE_SESSION_ID"`」
- 通过 `session send` 通知 CC 同样配置，CC 完成双向注册
- 双向 lookup 验证通过：mino → commander, commander → mino

**发现**：`myagents session send -p` 含换行符时 CLI exit 3 fail-fast，必须用 `--prompt-file`。

**发现**：`session send` 的 from 标签取自发送方 session title，不是 Agent 名称。若 title 是 auto-generated（如「娜娜 手机端独立机器人」），接收方看到误导标签。Fix：GUI 改 session title。

### 手机端独立机器人分析 (13:35-13:50)

CC 转发汤姆的问题：能否做手机端独立 bot 替代微信？同样模型下能更稳更快吗？
