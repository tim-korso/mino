## 系统卫生大清理

六层持久化机制完整逆向 + 全栈清理 + mac-hygiene 技能创建。

### 已清理

- **FlClashCore**: 100.9% → 0% CPU。根因是 `find-process-mode: always`（非 TUN stack——已经是 gvisor）
- **KeySound.app**: App + launchAgent + debug log 全删
- **夸克**: 22个配置开关优化 + Login Item 删除 + Quark Family 杀进程
- **WPS Family**: wpsoffice + wpscloudsvr + wpsbackend 全杀
- **commerce** (App Store): 16.5% CPU 关掉
- **Telegram**: 10.4% CPU 关掉
- **Pearcleaner**: 200MB 闲置内存释放
- **Xcode**: 闲置中关掉
- **LauncherAgents**: Google Updater×2 + MS Edge Updater + Sideloadly daemon → bootout + 删除 plist
- **LauncherDaemon**: Surge Helper + iBoysoft MagicMenu + Wireshark ChmodBPF → sudo bootout + 删除
- **邮件通知**: 三层关死（横幅+声音+角标+通知中心权限撤销）

### 发现

- **macOS 六层持久化模型**：父子进程监控(0) → Login Items(1) → LaunchAgents(2) → LaunchDaemons(3) → SMLoginItemSetEnabled(4) → System Extensions(5) → BGTaskScheduler(6)
- **WPS 三层复活**: 父进程监控(kqueue) + LoginItem(wpslaunchhelper.app) + UTI handler(docx关联)
- **Login Item vs App内开关是两套独立系统**: 夸克 App内关了 autoLaunchEnable，但 OS 级 Login Item 仍然注册
- **BGTaskScheduler 是最大盲区**: iOS 带过来的，macOS 无全局开关，用户完全不可见
- **孤儿数据**: MobileSync 9.8G, Qoder 628M, sideloadly 575M, playwright 1G — 可回收约 17G
