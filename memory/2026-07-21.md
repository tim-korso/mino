### 发现

- **macOS 六层持久化模型**：父子进程监控(0) → Login Items(1) → LaunchAgents(2) → LaunchDaemons(3) → SMLoginItemSetEnabled(4) → System Extensions(5) → BGTaskScheduler(6)

- **Login Item vs App内开关是两套独立系统**: 夸克 App内关了 autoLaunchEnable，但 OS 级 Login Item 仍然注册
- **BGTaskScheduler 是最大盲区**: iOS 带过来的，macOS 无全局开关，用户完全不可见

## 🧭 探索：Pearcleaner 自动化 → CLI=GUI 同引擎工具 → 三神器深度诊断

### 出发点
用户要求"深度调研 pearcleaner自动化"→ 发现 Pearcleaner CLI 和 GUI 共享同一 Swift AppPathFinder 引擎 → 引出更深的追问："还有哪些工具是这种模式？"→ 两轮调研（v3 失败，v5 成功）→ 发现三神器 → 全管线落地。

### 第一轮: Pearcleaner 深度调研

**关键发现**:
- CLI 7 子命令完整 (`pear list/list-orphaned/uninstall/uninstall-all/remove-orphaned/helper/ask-password`)
- 搜索敏感度 `defaults write settings.general.searchSensitivity -int 1` 即时生效——L1 保守只匹配 bundle ID，L2 对 Apple 系统 App 匹配整个 `com.apple.*` 命名空间 (Chess 3→3242 文件)
- 单二进制: ArgumentParser + SwiftUI 编译进同一 Mach-O。`AppPathFinder` 类在内存中只有一份编译产物——CLI 和 GUI 走同一条代码路径
- 产出: `mac-pearcleaner.sh`（自动切 L1 的安全封装）+ mac-hygiene v2 (Phase 5: 专业卸载层) + mac-chain 两条新链

### 第二轮: CLI=GUI 同引擎工具深度调研

**v3 失败** (wf_54e54679-637): 11 agents, 6671s, Synthesize agent 确定性 stall (6/11 dead)。根因: 合成 >3min 无文本输出 → 180s liveness check 触发。

**v5 成功** (wf_e7cd7e6e-adc): 5 agents, 398s, **0 dead**。改进: Synthesize 移出 Workflow → 主会话合成; effort='low'; search only。
- 125 个工具发现，82 个是真正新知
- 实测验证: powermetrics (NOPASSWD 已配置), bioutil (1 fingerprint, unlock+pay enabled), corebrightnessdiag (5 子命令全通), PlistBuddy (key-path 嵌套编辑), systemsetup (全部需 admin → 用 scutil/pmset 绕过), networkQuality (上行 589ms 偏高)

**教训**: Synthesize 不放进 Workflow 是硬规则。pipeline() 中一个 agent 死后不影响其他 item。`export` env var 不能通过 pipe 传递——必须用 `export VAR; echo | python3` 的格式。

### 第三轮: 长程任务 + Workflow 容错 + 三神器落地

**长程任务 v5**: Incremental 模式 (research-state.sh → state.json 持久化 → 缺口驱动)、Budget Pacing (四档自适应深度)、Goal Mode L1+L2 集成、API Fallback (api-router.sh 时段路由 + 健康检测)

**三神器封装** (`mac-deepdiag.sh`):

- `--plist`: key-path 嵌套读写 (binary plist → plutil convert → grep -aE → PlistBuddy)
- `--biometrics`: Touch ID 安全审计 (指纹数/解锁开关/ApplePay 开关)

**注册工具**: wf-recover + research-state → `myagents tool add` → PATH 自动注入 + 全 runtime 发现

**管线集成**: mac-security-audit (Phase 7.5 biometrics) + mac-observability (networkQuality)

### 四个悬案破解
1. **systemsetup**: 24 命令确认现存在于二进制——但全部需 admin。绕过: scutil/pmset/networksetup
2. **corebrightnessdiag**: 5 子命令全通——Night Shift JSON-like 状态可用
3. **plist 二进制**: plutil convert xml1 → temp → grep -aE 管道全通
4. **powermetrics**: NOPASSWD 已配置——脚本中 sudo -n 可用

### 堆栈修复
- `wf-recover.sh` init python3 -c 替换为 env-var heredoc (避免 bash 变量注入导致的 Python 语法错误)
- `mac-deepdiag.sh` power 模式: `export MD_HUMAN` 替代 pipe env (pipe 中断 env 传递)
- plist --keys: `grep -aE` 替代 `grep -E` (macOS BSD grep 的 "Binary file" 污染)
- bioutil 指纹计数: 正则从 `[0-9]\+` 改为 `[0-9]\+ biometric` (避免 UID 匹配)
- api-router.sh: 使用 `fable` 替代 `haiku` (haiku 模型已下架, 调用即报错)

### 核心洞见 (跨项目)
- **bash pipe 打断 env var**: `VAR=val cmd1 | cmd2` 只把 VAR 传给 cmd1, 不给 cmd2。用 `export`。
- **haiku 模型别名已漂移**: 在 moonshot provider 下 `haiku` → 已下架, 调用即报错。模型别名会随时间漂移——不要硬编码, 用探针验证。
- **99.3% 成功率**: 1373 agents / 105 workflows 的实证数据——Workflow 的可靠性远高于直觉。失败集中在 Synthesize 阶段而非搜索。
