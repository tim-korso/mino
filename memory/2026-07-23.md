## Windows 深度自动化中枢 — mino.ps1 建成

> 🧭 探索: Windows PowerShell×COM×AHK 三层自动化

**出发点**: 汤姆问 Windows 长盛不衰的原因 + 自动化能力。从 PowerShell 对象管道 → COM 跨应用 → AHK GUI 胶水, 拆了三层能力模型。汤姆说"要这种深度优化的自动化"。

**过程**:
1. 对比 macOS 13阶段管线, 发现 Windows 独有的优势: WMI/CIM结构化查询、COM完整对象模型、AHK Win32控件可达
2. 设计了统一中枢架构: `mino.ps1` 顶层入口 → 4个模块 → 3个共享库 → AHK胶水层
3. 全部从零建: core(日志/错误/快照) → wmi-helpers(启动审计/健康/性能/服务审查/电源/注册表异常) → com-helpers(COM生命周期防僵尸进程) → system(7命令) → cleanup(8命令) → office(Excel/Word/Outlook COM) → workplace(6命令) → AHK(热键+热字符串+微信胶水)
4. 踩坑: PS 5.1 不支持三元运算符 `?:`; 文件含 Unicode 字符(em-dash/中文引号)导致解析器级联崩溃, 全部重写为纯 ASCII; `$PSScriptRoot` 在不同调用模式下的行为不一致 → 统一由 mino.ps1 dot-source
5. 全模块端到端验证通过: system snapshot JSON输出准确, system registry检测到真实异常, Excel v16.0 COM全功能, workplace weekly 62 commits

**教训**:
- PS 5.1 的 parser 对文件编码极度敏感——任何非 ASCII 字符+非标准 BOM 都会级联破坏所有 dot-sourced 文件。纯 ASCII 是最小阻力路径。
- WMI/CIM 结构化查询 > 文本解析(Unix)。Windows 不需要像 macOS 那样手搓 `corebrightnessdiag` 挖未文档化 CLI——`Get-CimInstance Win32_*` 就是系统仪表化的标准入口。
- COM 对象必须显式追踪+释放——每一个 `New-Object -ComObject` 都是潜在的僵尸进程。`com-helpers.ps1` 的 `Use-ComApp` 包装器解决了这个问题。
- bash pipe 不跨 bash→powershell 边界——在 bash 里 pipe PowerShell 输出会失败。每个命令必须独立调用。

## Excel v3 — 高级 COM 功能 6 命令实测

> 🧭 探索: Excel COM 深度自动化 — 条件格式/数据验证/目标求解/迷你图/透视计算字段/工作表保护

**出发点**: 深度调研 Excel 自动化操作，精选复杂难题实现。WebSearch 扫了 8 个 COM 接口 → 选出 6 个高价值功能。

**过程**:
1. WebSearch 调研: 条件格式 (FormatConditions)、数据验证 (Validation)、目标求解 (GoalSeek)、迷你图 (SparklineGroups)、透视计算字段 (CalculatedFields)、工作表保护 (Protect/Unprotect)
2. 全部在 `office.ps1` 中实现: 6 个新函数 ~350 行，dispatch 路由扩展 8 行
3. 创建 `v3-test.xlsx` (9列×6行财务数据) 做测试载体
4. 逐命令实测，反复修复 COM 陷阱

**踩坑 (关键教训)**:
1. **PS 5.1 parser + BOM-less UTF-8 = 级联崩溃**: 所有 .ps1 文件必须带 UTF-8 BOM。中文 Windows 下 BOM-less 被读成 GBK → 中文注释破坏解析器 → 后续所有代码报 "Unexpected token"
2. **Edit tool 残留内容**: 编辑 `Write-ExcelHelp` 时 Edit 工具在函数闭包后留下了孤立行 (105-110行)，导致 parser 状态污染。不是编码问题，是工具边界问题
3. **Validation COM 状态腐败**: `.Validation.Delete()` 后 `.IgnoreBlank`/`.InCellDropdown` 必须在 `.Add()` 之后设置，否则 0x800A03EC。Delete→Add→SetProperties 顺序不可逆
4. **CalculatedFields PSMethod 碰撞**: `$pt.CalculatedFields` 当属性名与无参方法名相同时返回 `PSMethod` 而非 COM 对象。`$pt.CalculatedFields().Add()` (加括号) 先调用方法再拿返回值
5. **SparklineGroups.Add 维度匹配**: SourceData 行数必须等于 Location 行数，否则 COM 报错
6. **GoalSeek dispatch 参数拆分**: `$RestArgs -split '\s+', 4` 把 GoalValue+ChangingCell 合并成一个 arg → 需要二次 split
7. **protect 后 calcfield 失败**: 保护工作表后再操作透视表会拒绝访问 → 测试顺序必须先 unprotect 再 calcfield
8. **UserInterfaceOnly 不持久化**: `Worksheet.Protect(UserInterfaceOnly=$true)` 只在当前 COM 会话有效，保存后重开失效 → 需在 Workbook_Open 事件中重新设置

**6/6 全通**:
```
condfmt (databar green) → condfmt (colorscale red-yellow-green) → condfmt (iconset)
→ validate (list=选项1,选项2,选项3) → validate clear
→ goalseek (B10=150000 → B2: 12000→35500) ← 数值验证通过: 利润率=3500/35500=0.0986
→ sparkline (绿高红低标记) → calcfield (毛利率=利润/收入) ← 透视计算字段
→ protect (密码123) → 只读验证 → unprotect → kill (零僵尸)
```

**COM 端到端耗时**: 全部 6 命令 <30s (含 Excel 进程启动)。最慢的 GoalSeek ~5s (Excel 内部迭代求解)。

## PPT COM 自动化 v1 — 10 命令实测

> 🧭 探索: PowerPoint COM 深度自动化 — 创建/图表/过渡/动画/导出

**出发点**: 汤姆要求"深度调研 PPT自动化"，选挑战性高级功能实现。

**过程**:
1. WebSearch 调研: Application/Presentations/Slides/Shapes/Chart/SlideShowTransition/AnimationSettings/Slide.Export
2. 全部在 `office.ps1` 中实现: 10 个函数 ~400 行 + dispatch 路由
3. 逐命令实测，踩了 PPT COM 独特陷阱

**踩坑 (关键教训)**:
1. **AddChart2 需要可见窗口 (CRITICAL)**: `Presentations.Open($path, $false, $false, $false)` 第4参数 WithWindow=$false 阻止嵌入 Excel 图表引擎激活 → AddChart2 在管线中失败但独立脚本成功。根因: PowerPoint 图表由嵌入 Excel COM 引擎驱动，隐藏窗口时引擎不启动。Fix: `Open(..., $true)` + `DisplayAlerts = 2`。
2. **PpEntryEffect 枚举值巨大**: fade=1793, flyFromLeft=3331, pushRight=3853, wipeRight=2819, dissolve=1537, random=513。不是常规小整数。
3. **新建演示文稿 0 slides**: `Presentations.Add()` 不自动创建幻灯片 → 必须 `Slides.Add(1, 1)`。
4. **AnimationSettings 是旧版模型**: pre-2010 动画接口，EntryEffect 用同样的 PpEntryEffect 枚举。TextLevelEffect=1 实现逐级文本动画。
5. **SaveAs PDF ExportAsFixedFormat 失败**: MsoTriState 枚举类型不匹配，PS COM 无法传递正确的类型对象。
6. **ChartData.Workbook 操作受限**: headless 模式下 CSV 数据加载失败，使用默认图表数据。

**9/10 全通** (video 命令编码但未测试):
```
new (4 slides) → slide (layout=content) → text (title+body) 
→ image → chart (native column, 嵌入 Excel 引擎) 
→ transition (push+advance 3s) → animate (flyin+fade) 
→ export-slide (1920×1080 PNG) → layout (retro+themes) → save (pptx)
```

**已知限制**:
- `save pdf`: ExportAsFixedFormat MsoTriState 类型问题，pptx 保存正常
- `video`: CreateVideo 已编码但未测试 (耗时长)
- Chart 数据: headless 下用默认数据，不支持 CSV 自定义数据加载
