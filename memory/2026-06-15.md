## 🧭 探索：B站视频转录 — Agent 看视频的管道

**出发点**
需要让 Agent 能"看"B站视频内容——把视频里的语音转成文字，方便后续分析、验证、引用。B站视频没有官方下载 API，反爬也严。

**过程**
1. **yt-dlp 直下** → 412 阻断。B站服务端反爬直接拒了，此路不通
2. **AI 字幕** → 这台 Mac 上的视频没有。依赖视频上传者开，不可控
3. **本地 Whisper** → 没装。装要走 brew + 模型下载，太重，暂绕
4. **突破 — Playwright 浏览器拦截** → 在真实浏览器里播放视频，监听网络请求抓 .m4s 音频流的 URL。浏览器环境 = 真实用户，绕过了服务端反爬
5. **下载** → curl 带 Referer + UA 头下载 .m4s 音频片段
6. **转码** → ffmpeg 转 16kHz mono wav（SiliconFlow API 要求）
7. **切片** → 5 分钟一段（API 有长度限制）
8. **转录** → SiliconFlow SenseVoiceSmall API 逐片转写 → 拼接全文

**教训**
- 视频站的防线在服务端（请求头校验），不在浏览器。Playwright 浏览器内拦截是关键突破
- 转录瓶颈不在模型速度在下载 IO——如果做批量，并发下载是核心收益
- B站 .m4s 音频流 URL 有时效，抓到要立刻下
- 管道文档已落 `.claude/skills/claim-verification/references/video-pipeline.md`
